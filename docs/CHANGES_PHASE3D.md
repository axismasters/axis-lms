## 추가 개선 (v3-r3)

**버전**: v3-r3 — v3-r2는 GitHub 업로드 가능 상태로 확정. 이번 라운드는 5개 추가 개선
요청에 대응한다. 이 섹션이 최신 내용이며, v3-r2 이하 내용은 아래에 각각 보존한다.

### 1. 학부모 페이지 Rival/Emblem/SP/Tier 완전 제거(코드+주석+문서)

v3-r1/r2에서 실제 UI와 대부분의 주석은 이미 정리되어 있었지만, `docs/PARENT_PAGE_
CONSTITUTION.md`의 원칙 6번 본문이 여전히 "Rival 대결·Emblem·SP" 지표명을 한 번 나열하고
있었다. 이번엔 이 부분까지 완전히 추상화해서 어떤 지표명도 등장하지 않도록 다시 썼다
("학생 동기부여를 위한 게임형 경쟁·수집 요소(대결 전적, 수집형 배지, 포인트 점수 등)"
처럼 일반 명사로만 서술).

`docs/PARENT_PAGE_CONSTITUTION.md`의 "라이벌 상대방 식별 정보는 노출하지 않는다 —
**Tier까지만 확인 가능**" 문구를 완전히 삭제했다. 이 문구는 애초에 학생용 Rival 화면
정책을 설명하려던 참고 각주였는데, 학부모 페이지 헌법 문서에 있는 게 맞지 않았다(정책
자체는 `StudentRival.tsx` 자체 주석에 이미 정확히 남아있어 삭제해도 정보 손실이 없다).

성장 리포트(`ParentGrowthReport.tsx`)를 4가지 방향으로 강화했다:

- **목표 대비**: 과목별 보완 필요도 막대 아래 "목표(90%)까지 N%p" 표시.
- **과목별 변화**: 과목별 응시 이력을 앞/뒤 절반으로 나눠 "이전 대비 ▲/▼ N%p" 계산·표시
  (표본 2건 미만은 계산하지 않음).
- **주간 변화**: 리포트 탭에 지난주 대비 이번 주 출결율/숙제완료율 증감(%p) 표시.
- **상담용 요약**: 테스트 평균·직전 대비 변화·보완 필요 과목·이번 달 결석·숙제 완료율을
  한 문단 문장으로 조합한 카드 신규 추가 — 학부모가 상담 전에 미리 읽고 갈 수 있는
  요약이다.

새로 반영한 아이디어와 아직 반영하지 못한 아이디어는 `docs/PARENT_PAGE_ENGAGEMENT_
IDEAS.md`에 "v3-r3 신규 반영" 섹션으로 정리했다(5개 항목).

정리되어 있었고, 이번 재검색에서 새로 발견된 위반은 없었다 — 남아있는 유일한 예외는

### 5. 시험지별 그래프 표현 강화 — 문항별 정답률 신규

`TeacherExamScores.tsx`에 "문항별 정답률" 막대그래프를 신규 추가했다. 담당 학생 중
채점완료된 제출의 `AnswerRecord`를 문항별로 집계해 정답률(%)을 계산한다. 자동채점
문항은 `isCorrect` 값을 그대로 쓰고, 수동채점 문항은 "만점 획득 = 정답"으로 근사
처리했다(부분점수를 받은 경우는 오답으로 집계 — 단순화된 근사치임을 코드 주석에
명시). `gradeSubmissionByTeacher`(총점 직접 입력) 경로로 채점된 제출은 문항별 점수
데이터가 없어 이 그래프에 집계되지 않는다 — 데이터가 있는 문항만 표시하고, 문항별
데이터가 하나도 없으면 이 섹션 자체를 숨긴다(빈 그래프를 보여주지 않음).

학생/학부모 화면은 이미 v3-r1 이전부터 점수추이·평균대비·IF점수추이·놓친점수누적·
IF이유비율·시험군내성장변화까지 6종 그래프를 갖추고 있어(Phase 3B), 이번엔 유일하게
빠져있던 "문항별 정답률"을 선생님 화면에 보강하는 것으로 충분하다고 판단했다.

## §GPT(개발 총괄)에게 전달할 의견 (v3-r3 추가분)

1. **`TeacherExamGrading.tsx`의 "채점" 타이틀은 세 번째 라운드에도 수정하지 못했다.**
   반복 지적되는 부분이니 정말 통일이 필요하면 이 파일 하나만 불변 목록에서 제외하는
   결정을 부탁드린다 — 다른 모든 곳은 이미 "내 시험지"/"내 시험지 관리" 흐름으로
   통일되어 있다.
2. **문항별 정답률은 "만점 획득 = 정답"으로 근사 처리했다.** 부분점수를 받은 문항이
   실제로는 "완전히 틀린 게 아니라 일부만 맞았다"인데 이 그래프에서는 오답으로
   집계된다. 서술형처럼 부분점수가 자주 나오는 문항 유형이 많다면, "부분정답" 상태를
   별도로 구분하는 3단계(정답/부분정답/오답) 그래프로 확장하는 게 더 정확할 것 같다.
3. **과목별 "목표 대비" 90%는 코드 상수로 고정되어 있다.** 실제로는 학년·과목별로
   다른 목표 기준이 필요할 수 있어, 다음 라운드에서 관리자가 과목별 목표치를 설정할
   수 있게 만드는 것도 고려해볼 만하다.

## 반려 대응 (v3-r2)

**버전**: v3-r2 — v3-r1 반려 사유 6개 전부 반영. 이 섹션이 최신 내용이며, v3-r1 이하
내용은 아래에 각각 보존한다.

### 1. 테이블 고정 구조 추가 수정

`StudentList.tsx`, `AttendanceStatus.tsx`의 **메인 데이터 테이블**(요약 카드가 아니라
실제 목록 표)이 아직 구 `.axis-table-wrap` 패턴을 쓰고 있었다. v3-r1 때는 이 두 화면의
요약 카드만 필터 버튼으로 바꿨고, 메인 표는 손대지 않았었다(item 6의 8개 대상 목록에
관리자 학생목록/출결현황의 메인 표가 명시적으로 포함되어 있지 않았기 때문) — 이번에
명시적으로 지적받아 두 표 모두 `.axis-table-scroll`(wrapper `max-height: 620px` +
`overflow: auto`, `thead th`가 wrapper 내부 스크롤 기준 `top: 0`에 고정, `border-collapse:
separate` 강제)로 전환했다. 헤더/본문 컬럼 폭은 같은 `<table>` 안에 있으므로 브라우저의
기본 표 레이아웃 알고리즘이 항상 동기화한다(별도 처리 불필요).

### 2. 관리자 시험 목록 클릭 UI 수정

`AssessmentList.tsx`: `<tr>`의 `cursor-pointer` 클래스와 `onClick={() =>
navigate(...)}`를 제거했다. 대신 마지막 "관리" 컬럼의 "상세" 칩(장식용 span)을 실제
`<Button variant="outline" size="sm">상세 보기</Button>`로 바꾸고, 이 버튼에만 클릭
핸들러를 달았다. hover/cursor-pointer/focus-visible은 이제 버튼 자체(shadcn `Button`
컴포넌트가 기본 제공)에만 적용된다 — 행의 다른 셀(시험명, 응시현황 등)은 더 이상 클릭
가능한 것처럼 보이지 않는다.

### 3. 선생님 페이지 문구 전수 정리

지시된 6개 표현을 정확히 교체했다:

| 구 표현 | 신 표현 | 위치 |
|---|---|---|
| 최근 성적 | 최근 테스트 결과 | TeacherHome.tsx, TeacherStudentDetail.tsx |
| 성적 보기 | 학생별 성적 보기 | TeacherHome.tsx |
| 성적 데이터 | 테스트 결과 데이터 | TeacherStudentDetail.tsx, TeacherGrades.tsx |
| 담당 학생 성적 확인 | 담당 학생 테스트 결과 확인 | TeacherGrades.tsx, TeacherRoutes.tsx |
| 수업노트 바로가기 | 수업자료에서 수업노트 확인 | TeacherStudentDetail.tsx |
| 수업노트 작성/확인하기 | 수업자료 열기 | TeacherStudentDetail.tsx |

"수업노트 작성/확인하기" 링크가 가리키던 구 경로 `/teacher/notes`도 `/teacher/materials
?tab=notes`로 함께 갱신했다(리다이렉트가 있어 기존에도 깨지지는 않았지만, 새 화면으로
바로 연결되게 정리). `TeacherLayout.tsx`의 `isActive` 판정 로직에 남아있던 구
`/teacher/videos` 관련 죽은 코드와 "채점 탭"이라는 낡은 주석도 함께 정리했다.

`채점하기`/`채점완료`/`미채점`처럼 실제 채점 행위 자체를 뜻하는 표현은 지시대로 그대로
뒀다. 전체 재검증 결과 화면 제목·메뉴명으로 단독 "채점"이 남아있는 곳은
`TeacherExamGrading.tsx`(불변 파일) 단 한 곳뿐임을 확인했다(§GPT 전달 의견 참조).

### 4. 학부모 페이지 문구/주석 정리

실제 UI 코드에는 v3-r1에서 이미 Rival/Emblem/SP가 제거되어 있었지만, **개발자 주석과
정책 문서에 여전히 이 지표명이 나열식으로 반복**되고 있었다. 이번에 다음을 정리했다:

- `ParentLayout.tsx`: "자녀 성장(Emblem/SP/Tier) 확인 흐름" → "자녀 성장 리포트(테스트
  변화·출결 흐름·대학추천 요약)"로 교체.
- `ParentHome.tsx`: 헌법 원칙 목록에 **"6. 학생용 게임형 지표는 학부모 화면에 노출하지
  않는다"**를 신설해 추가하고, 본문 곳곳의 "Rival/Emblem/SP" 나열식 주석 4곳을 이
  원칙 하나로 통일해서 표현하도록 정리.
- `ParentGrowthReport.tsx`: 헤더 주석 정리.
- `docs/PARENT_PAGE_CONSTITUTION.md`: **5개 원칙 → 6개 원칙으로 갱신**(신규 6번 원칙
  추가), 기존 1·2번 원칙 본문에 남아있던 "Tier/Emblem/SP" 나열도 정리(사실 이 문구는
  v2 시점에 쓰인 것이라 v3-r1의 노출 금지 정책과 모순되고 있었다 — 이번에 정책과
  일치하도록 내용 자체도 고쳤다, 단순 표현 순화가 아니라 정확성 수정).
- `docs/PARENT_PAGE_ENGAGEMENT_IDEAS.md`: 도입부 원칙 문구 정리.
- **과거 라운드 기록(CHANGES/MODIFIED_FILES/QA 문서의 v3-r1 이하 섹션)은 의도적으로
  그대로 뒀다** — "무엇을 제거했다"는 이력을 설명하려면 그 이름을 언급할 수밖에 없어,
  이 부분까지 지우면 오히려 과거 작업 내용을 알 수 없게 된다. 현재 정책을 나타내는
  문서와 소스 코드만 정리 대상으로 삼았다.

### 5. GitHub 업로드 zip 구조 명확화

v3-r1까지 `-github-upload.zip`을 "v1 베이스라인 대비 누적 diff"로 만들었는데, 이 방식은
①삭제된 파일(`StudentFinance.tsx`)을 diff로 표현할 수 없고 ②패치용인지 전체 업로드용인지
zip만 봐서는 불명확하다는 문제가 있었다. **v3-r2부터 `-github-upload.zip`은 diff가 아니라
프로젝트 전체 구조를 담은 완전한 패키지로 전환**했다. `docs/APPLY_ORDER_PHASE3D.md`에
적용 방법(저장소 통째로 교체 / 새 저장소로 업로드)과, 그래도 부분 패치를 원하는 경우의
주의사항(삭제 파일 수동 처리 필요)을 모두 명시했다.

### 6. 테이블 헤더/컬럼 고정 구조 재작업

기존 `.axis-th-sticky`(top: 56px/52px, 페이지 스크롤 기준) 방식은 `.axis-table-wrap`이
`overflow-x`만 설정하고 `overflow-y`는 미설정이라 브라우저가 "가장 가까운 스크롤
컨테이너"를 모호하게 판정할 수 있는 문제가 있었다. 신규 `.axis-table-scroll` 패턴
(`src/index.css`)은 wrapper 자체에 명시적 `max-height` + `overflow: auto`를 줘서
wrapper 하나가 수평/수직 스크롤을 모두 담당하는 유일한 컨테이너가 되게 했다. 헤더는
이제 `top: 0`(wrapper 내부 스크롤 기준)에 고정되며, `border-collapse: separate`를
강제해(Tailwind `border-collapse` 클래스보다 선택자 특이성이 높아 자동 우선 적용)
sticky th의 하단 border/box-shadow가 스크롤 중 사라지는 버그도 함께 막았다.

8개 대상 전부 적용 완료: 엠블럼 관리, 성장관리 학생 목록 + SP 지급 이력(2개 표),

양쪽 모두에 자동 적용된다.

- **기본 템플릿 2종 추가** (`buildSuneungTemplate()`, `buildNaeshinTemplate()`):
  - **수능형**: 30문항. 1~3번 2점 / 4~13번 3점 / 14~22번 4점 / 23~30번(선택과목) 4점.
    ⚠️ **주의(원장님 확인 필요)**: 이 배점 구조를 문항 수 그대로 반영하면 합계가
    6+30+36+32 = **104점**이 되어, 원 지시서의 "총점 100점" 문구와 정확히 맞지 않는다.
    실제 수능 수학은 공통 22문항 + 선택 8문항 배점이 이보다 더 세분화되어 있어 100점에
    맞춰져 있는데, 이번 지시서에 명시된 4개 구간 배점을 그대로 적용하면 104점이 나온다.
    현재는 **지시서에 적힌 배점 구간을 그대로** 구현해 두었고(임의로 배점을 줄이지
    않음), 각 문항의 배점은 문항 구성 화면에서 자유롭게 수정 가능하다. 정확히 100점
    구조를 원하시면 다음 라운드에서 구간을 조정하겠다.
  - **내신형 대시**: 24문항, 100점 자동 배분(100 = 4점×20문항 + 5점×4문항 — 나머지 4점을
    앞쪽 4개 문항에 1점씩 더 배분).
  - 템플릿 적용 시 문항 type은 모두 '객관식' 기본값(정답은 비워둠 — 교사가 이후 채움).
    교사가 "+ 문항 추가"를 30번 누르지 않아도 기본 구성이 즉시 생성된다.
  - 이미 입력한 정답이 있는 상태에서 템플릿을 누르면(기존 구성을 덮어쓰므로) 확인 모달을
    띄운다.
- **문항 추가/삭제 스테퍼**: "총 N문항 · 만점 M점" 옆에 좌측 "−"(마지막 문항 삭제) / 우측
  "+"(문항 추가) 버튼 구조 추가.
  - 최소 문항 수(`MIN_QUESTIONS = 1`) 아래로는 삭제 불가(버튼 비활성화 + 안내 토스트).
  - 삭제 대상 문항에 입력된 정답이 있으면(입력값 손실 가능성) 확인 모달을 띄우고, 없으면
    바로 삭제.
  - 문항별 개별 삭제(휴지통 아이콘)도 동일한 확인 로직을 타도록 통합(`requestRemoveQuestion`).
- **버그 수정(부수적 발견)**: 이 확인 모달에 `AlertDialog` 컴포넌트를 재사용하는 과정에서,
  `AlertDialog`가 `Dialog` 안에 중첩될 때 확인 팝업의 배경(오버레이)을 클릭하면 클릭
  이벤트가 상위 `Dialog`까지 버블링되어 시험 등록 모달 전체가 함께 닫히는 기존 버그를
  발견했다. `src/components/ui/alert-dialog.tsx`의 배경 클릭 핸들러에 `stopPropagation()`을
  추가해 수정(다른 화면에서 `AlertDialog`를 쓰는 곳에도 동일하게 적용되는 일반적인 수정).

# Phase 3D v3-r5 — teacher exam structure cleanup

기준: v3-r4-r1(GitHub 업로드 승인됨). 이번 라운드는 선생님 화면의 "시험 구조"를 정리한다 —
시험 출제/문항관리/채점관리(문항 기반 Exam)와 성적 입력 자료(내신/전국연합/수능실전모의고사
점수 기록)를 명확히 분리한다. 커밋 요약: `Phase 3D v3-r5 teacher exam structure cleanup`.

## 1. 시험 종류 카탈로그 정리 (`src/lib/assessmentData.ts`)

- `EXAM_CATEGORIES`에서 `national-mock`(전국모의고사)·`school-record`(실제내신성적)·
  `weekly-suneung`(수능실전주간루틴) 3개를 완전히 제거했다. 전수 조사 결과 이 3개는
  단 하나의 실제 시험(Exam)에도 categoryId로 쓰인 적이 없었다(순수 카탈로그상의 죽은
  항목) — 제거해도 기존 공개 성적 라벨 표시에 영향 없음을 확인.
- 남은 5개: `entrance-test`(입학테스트, 관리자 전용) · `unit-eval`(단원평가) ·
  `certification`(인증평가, 관리자 전용) · `mock-school`(내신대비모의고사) ·
  `mock-suneung`(수능실전모의고사, 성적 입력 자료로 재분류 — 기존 데이터 라벨링용으로만
  카탈로그 유지, 신규 생성 대상 아님).
- 신규 상수 2개 추가:
  - `TEACHER_CREATABLE_EXAM_CATEGORY_IDS = ['unit-eval', 'mock-school']` — 강사가
    "내 시험지 관리"에서 만들고 채점하는 문항 기반 시험만.
  - `ADMIN_CREATABLE_EXAM_CATEGORY_IDS = ['entrance-test', 'unit-eval', 'certification', 'mock-school']`
    — 관리자도 수능실전모의고사는 문항 기반으로 새로 만들지 않는다(성적 입력으로 이관).

## 2. `src/lib/phase2dData.ts` — weekly-suneung 제거

- `GRADE_TABS`의 'suneung' 탭 `categoryIds`에서 `weekly-suneung` 제거(`['mock-suneung']`만 남음).
- `EXAM_TYPE_MAP`에서 `weekly-suneung` 항목 삭제.
- `national-mock`/`school-record`는 그대로 유지 — 이 둘은 Assessment Engine과 무관하게
  `teacherSchoolRecordInput.ts`/`teacherMockExamInput.ts`(성적 입력) → `PHASE2D_SCHOOL_RECORDS`/
  `PHASE2D_NATIONAL_MOCKS`를 거쳐 학생 성적 탭(실제내신/전국모의)에 표시되는 정상 경로이며,
  시험지 생성(AssessmentFormModal) 대상이었던 적이 없다.

## 3. 시험 종류 선택지 역할 분리 (`src/components/AssessmentFormModal.tsx`)

- `EMPTY_FORM.categoryId` 기본값을 `EXAM_CATEGORIES[0].id`(구 기본값이 우연히
  'entrance-test'였음)에서 `'unit-eval'`로 변경(관리자/교사 모두 유효한 카테고리).
- 시험 종류 `<Select>`가 이제 `EXAM_CATEGORIES` 전체 대신 역할별 `categoryOptions`
  (교사: `TEACHER_CREATABLE_EXAM_CATEGORY_IDS`, 관리자: `ADMIN_CREATABLE_EXAM_CATEGORY_IDS`)만
  보여준다. 모달이 열릴 때 `categoryId`도 그 역할의 첫 옵션으로 재설정된다.

## 4. 교사 시험 목록/채점 화면에서 제외 (defense-in-depth 이중 방어 포함)

## 6. "수능실전주간루틴" 명칭 제거 — 화면 노출명 정리

`수능실전주간루틴`은 개발 설명용 내부 개념일 뿐 화면에 노출되는 이름이 아니어야 한다는
지시에 따라, 화면에 보이는 모든 지점을 "수능실전모의고사 입력/결과"로 정리했다.

- `StudentMockExams.tsx`(학생 테스트 메뉴) / `ParentMockExams.tsx`(학부모 시험 항목) —
  "수능실전 주간 루틴 바로가기" 진입 카드를 완전히 제거. 미사용 `Link`/`ChevronRight`
  import도 함께 정리.
- `StudentWeeklyMocks.tsx` / `ParentWeeklyMocks.tsx` — 화면 타이틀을 각각
  `"수능실전 주간 루틴"` → `"수능실전모의고사 결과"`, `"자녀 수능실전 주간 루틴"` →
  `"자녀 수능실전모의고사 결과"`로 변경. 라우트 경로(`/student/weekly-mocks` 등)와
  파일명은 내부 코드명일 뿐 사용자에게 노출되지 않으므로 유지(진입 카드가 제거되어
  직접 URL로만 도달 가능 — 별도 시험 종류가 아니라 mock-suneung 결과의 회차별 누적
  조회 화면이라는 원래 설계 의도 그대로 유지, 데이터/로직 변경 없음).
- `teacherMockExamInput.ts`의 `getMockExamLabel()` — 저장값 `examType`(내부 분류값,
  변경하지 않음)을 화면에 표시할 때만 `'수능실전'` → `'수능실전모의고사'`로 풀어서
  보여주는 `displayExamType()` 헬퍼 추가(학생 화면 `StudentTargetPreview.tsx`에 그대로
  반영됨).
- `TeacherUniversityData.tsx` — 수능실전모의고사 입력 탭의 성적표 헤더(`"수능실전 성적표
  입력"` → `"수능실전모의고사 성적표 입력"`)와 읽기전용 "시험 종류" 표시(`"수능실전"` →
  `"수능실전모의고사"`) 텍스트 정정. 저장 로직(`examType: '수능실전'` 내부값)은 변경하지
  않음 — 화면 표시 문자열만 수정.
- `StudentRoutes.tsx`/`ParentRoutes.tsx`의 라우트 주석도 새 명칭 정책에 맞게 갱신
  (코드 주석이라 사용자에게 노출되지는 않지만 일관성을 위해 정리).
