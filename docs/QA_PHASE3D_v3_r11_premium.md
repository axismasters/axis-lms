## C. 리스크 / 한계

- `AxisEmblemBadge` 내부 좌표/그라데이션 로직이 대폭 바뀌었지만 **prop 시그니처는 100% 동일**하여
  호출부 코드는 건드리지 않았다 — 회귀 위험은 "시각적으로 의도한 대로 보이는지"에 집중되며,
  타입 레벨 위험은 낮다(오프라인 하네스로 확인됨).
- `AxisEmblemPlaque`는 순수 프레젠테이션 컴포넌트(상태/데이터 없음)라 로직 리스크가 거의 없다.
- 대표 엠블럼 선정 로직은 클라이언트에서 매 렌더 계산(정렬)하는 방식이라, 엠블럼 카탈로그가
  수백 개 단위로 커지면 메모이제이션을 고려할 수 있으나 현재 카탈로그 규모(수십 개)에서는
  문제 없다.
- 최종 그린 빌드는 GitHub Actions에서 확정 필요(오프라인 하네스는 회귀 판별용, 완전한
  `@types` 검증은 아님).
