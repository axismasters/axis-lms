# PARENT_PAGE_CONSTITUTION.md — 학부모 페이지 헌법

Phase 3D v2에서 확정, v3-r1에서 6번째 원칙 추가. AXIS LMS의 학부모(보호자) 화면을
만들거나 수정하는 모든 작업은 이 6개 원칙을 따른다. (`src/pages/parent/ParentHome.tsx`
상단에도 동일한 원칙이 요약 주석으로 반영되어 있다.)

## 1. 학부모 페이지는 "확인 화면"이 아니라 "자녀 성장을 계속 들여다보고 싶은 화면"이다

학부모가 이 화면에 최대한 오래 머무르고, 자녀의 성장 기록·테스트 변화·IF 분석·출결 흐름·
목표대학추천 흐름을 이것저것 눌러보며 확인하고 설득될 수 있게 만든다. 단순히
숫자를 나열하는 대시보드가 아니라, 클릭할수록 더 알고 싶어지는 화면을 지향한다.

## 2. 중심은 납부 확인이 아니라 자녀 성장 확인이다

- 수납/미납 정보는 보조 정보로 낮춘다. 홈 화면 최상단에 수납 정보를 배치하지 않는다.
- 총 청구액·총 납부액 같은 총액성 금액은 요약 카드에 표시하지 않는다. 미납이 있으면
  빨간 계열 배지로 "미납 있음", 없으면 초록 계열 배지로 "미납 없음"만 표시한다.
- 반대로 성장 리포트(테스트 변화·출결 흐름·대학추천 요약)는 홈 화면 상단부에 눈에 띄게
  배치한다.

## 3. 클릭 가능한 카드/요약/상세 진입 요소는 버튼/카드 버튼처럼 명확히 보이게 만든다

호버 시 그림자·테두리 변화(`axis-card-clickable`)를 적용해 "눌러도 되는 것"과 "그냥
텍스트"를 시각적으로 분리한다. 클릭 불가능한 정보 카드에는 이 스타일을 적용하지 않는다.

## 4. 학부모는 확인자다 — 입력/수정 기능은 어떤 화면에도 넣지 않는다

성적 직접 입력, 상담 기록 작성, 납부 등록/환불 요청, 프로필 수정 등 어떤 형태의
입력·수정 기능도 학부모 화면에 추가하지 않는다. 모든 학부모 화면은 읽기 전용이 원칙이다.

## 5. 상담 기록 원문은 학부모에게 노출하지 않는다

선생님이 작성하는 상담 기록(상담일/유형/대상/내용/작성자)은 내부 기록이며, 학부모/학생
화면 어디에도 원문을 노출하지 않는다. 목표대학추천 요약 화면 등에서 "상담은 선생님에게
문의하세요"처럼 안내만 하는 것은 허용되지만, 상담 내용을 직접 보여주는 카드나 페이지는
만들지 않는다. 공개용으로 가공된 상담 리포트가 필요하다면 별도의 승인/가공 절차를 거친
새로운 기능으로 다뤄야 하며, 원문을 그대로 노출하는 방식으로 우회하지 않는다.

## 6. 학생용 게임형 지표는 학부모 화면에 노출하지 않는다

Rival 대결·Emblem·SP 같은 학생 동기부여용 게임형 UI는 학생 전용이다. 학부모 화면에는
이런 지표를 배지나 아이콘, 포인트 숫자 형태로 직접 보여주지 않는다. 대신 그 배경이 되는
실제 데이터(테스트 결과 변화, 출결 흐름, 대학추천 변화, 학습 리포트)를 학부모가 이해하기
쉬운 문장·그래프 형태로 재구성해서 보여준다. 개발 주석이나 문서를 쓸 때도 "Rival/Emblem/
SP 노출 금지"처럼 구체적 지표명을 반복 나열하기보다, 이 원칙 하나로 통일해서 표현한다.

## 함께 지켜야 하는 기존 AXIS 확정 원칙 (참고)

- 합격률/합격 가능성/합격 보장/안정 합격/불합격 표현 금지.
- 라이벌 상대방 식별 정보(닉네임/연결 관계 상세)는 노출하지 않는다 — Tier까지만 확인 가능.
- IF는 별도 메뉴가 아니라 "테스트 성적표 상세" 안에서만, 조회 전용으로 확인 가능.
