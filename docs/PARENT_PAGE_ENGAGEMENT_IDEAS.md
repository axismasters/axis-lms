# PARENT_PAGE_ENGAGEMENT_IDEAS.md

Phase 3D v3-r1 추가 요구사항(학부모 페이지 체류 시간·설득력 강화) 대응 문서.
Claude가 제안하는 아이디어를 정리하고, 실제 반영 여부를 표시한다.

⚠ 원칙: 어떤 아이디어도 학생용 게임형 지표를 학부모 화면에 직접 노출하지 않는다.
학부모에게는 객관적 데이터(테스트 결과/출결/대학추천/학습 리포트)를 "성장 리포트"
형태로만 보여준다.

---

## 실제 코드에 반영한 항목

1. **탭 구조(테스트/출결/목표대학/리포트)** — 한 화면 안에서 눌러볼 거리를 늘리기 위해
   ParentGrowthReport.tsx를 4개 탭으로 재구성. (`src/pages/parent/ParentGrowthReport.tsx`)
2. **기간 필터(이번 달/최근 3개월/전체)** — 테스트·출결 탭에 적용, 스스로 원하는 범위를
   좁혀볼 수 있게 함.
3. **점수 변화 추이 그래프(SVG 선 그래프)** — 최근 5회 테스트 % 점수를 순수 SVG로 시각화
   (외부 차트 라이브러리 없이 구현, `TrendSparkline` 컴포넌트).
4. **과목별 보완 필요도 막대 그래프** — exam.subject 기준 평균 % 오름차순 정렬(`HBar`).
5. **IF 이유 3종 비율 스택 막대** — `getIfCumulativeSummary()`(기존 studentIfRecord.ts
   함수)를 재사용해 계산 실수/개념 부족/시간 부족 비율을 시각화(`ReasonRatioStack`).
6. **시험 상세 클릭 → 모달** — 카드 클릭 시 실제점수/반평균/최고점 막대 비교 + 놓친 문항
   번호 목록을 보여주는 상세 모달(`TestDetailModal`). 시험지별 그래프 표현 강화 요구사항과
   함께 처리.
7. **자녀 선택(다자녀 지원)** — 기존 ParentHome.tsx 패턴을 재사용해 탭 상단에 배치.
8. **홈 화면 "이번 달 변화" 요약 카드** — 성장 리포트 상단에 테스트 기록 건수/최근 한 달
   평균/보완 필요 과목 3개 요약 카드.
9. **주간/월간 학습 리포트** — 실제 출결·숙제 완료율·테스트 평균을 최근 7일/30일 기준으로
   집계(가짜 데이터 아님 — 기존 AttendanceContext/HomeworkContext/AssessmentContext에서
   직접 계산).
10. **선생님 공개 코멘트 기능 신규 구축** — `src/lib/parentComments.ts` 신규 모듈 +
    `TeacherStudentDetail.tsx`에 작성 UI 추가. 상담 기록(내부용) 원문이 아니라 선생님이
    학부모용으로 직접 다시 쓴 문장만 저장·노출한다.
11. **목표대학 탭에서 기존 화면으로 연결** — `/parent/target-summary`(이미 완성되어 있던
    화면)로 자연스럽게 이어지는 진입 카드로 처리해 로직 중복을 피함.
12. **문장형 해석** — "최근 테스트 점수가 N%p 올랐습니다"처럼 숫자 대신 읽기 쉬운 문장으로
    변화를 설명.
13. **ParentHome.tsx 성장 리포트 진입 카드에서 Tier/Emblem/SP 표현 완전 제거** — 대신
    "테스트 변화 · 출결 흐름 · 대학추천" 문구로 교체하고, `useGrowth()`/`growthData.ts`
    의존을 이 파일에서 아예 제거함.

### v3-r3 신규 반영

14. **과목별 "목표 대비" 격차 표시** — 과목별 보완 필요도 막대 아래에 "목표(90%)까지
    N%p" 형태로 목표 기준 격차를 추가 표시. 목표 퍼센트는 현재 코드 상수
    (`SUBJECT_TARGET_PCT = 90`)로 고정되어 있다 — 실제로는 학생/학년/과목별로 다른
    목표를 쓸 수도 있어 다음 라운드에서 조정 가능하게 만드는 게 좋을 것 같다.
15. **과목별 "변화" 표시(이전 대비)** — 기존에는 과목별 평균이 "현재 스냅샷"만
    보여줬는데, 각 과목의 응시 이력을 앞/뒤 절반으로 나눠 "이전 대비 ▲/▼ N%p"를 함께
    보여주도록 확장. 표본이 2건 미만인 과목은 계산하지 않는다(데이터 왜곡 방지).
16. **주간 변화(전주 대비)** — 리포트 탭의 "주간 학습 리포트" 카드에 지난주(8~14일 전)
    대비 이번 주 출결율/숙제완료율 증감을 함께 표시.
17. **상담용 요약 카드 신규** — 리포트 탭에 "상담용 요약" 카드를 추가해, 테스트 평균·
    직전 대비 변화·가장 보완이 필요한 과목·이번 달 결석 횟수·숙제 완료율을 한 문단
    문장으로 조합해서 보여준다. 학부모가 상담 전에 미리 읽고 갈 수 있는 요약 카드
    컨셉이다.
18. **`PARENT_PAGE_CONSTITUTION.md`에서 "Tier까지만 확인 가능" 문구 삭제** — 이 문구는
    원래 학생용 Rival 화면 정책을 설명하려던 참고 각주였는데, 학부모 페이지 헌법
    문서에는 애초에 맞지 않는 내용이라 완전히 제거했다(해당 정책은
    `StudentRival.tsx` 자체 주석에 이미 정확히 기술되어 있어 정보 손실은 없다).

## 아직 반영하지 못한 항목

1. **최근 3회/최근 5회 "비교" 전용 뷰** — 현재는 "추이 그래프"(연속 값)로만 보여주고,
   두 시점을 나란히 놓고 비교하는 전용 UI(예: 이전 시험 vs 이번 시험 막대를 좌우로 배치)는
   아직 만들지 않았다. 다음 라운드에서 `TestDetailModal`에 "직전 시험과 비교" 섹션을
   추가하면 좋을 것 같다.
2. **월간 리포트의 "전월 대비" 비교** — v3-r3에서 **주간(전주 대비)은 구현했지만**, 월간
   수치(테스트 평균/숙제 완료율/결석 횟수)의 "지난달 대비" 증감은 아직 계산하지 않았다.
   지난달 데이터 범위를 별도로 뽑아서 동일한 방식으로 비교하면 될 것 같다.
3. **수업자료 열람 현황을 성장 리포트 안에 통합** — 현재 수업자료는 학부모 홈에서 별도
   섹션으로만 노출되고(v2), 성장 리포트의 "리포트" 탭 안에는 아직 연결하지 않았다.
4. **문항별 정오표를 더 시각적으로(그리드/색상 매트릭스)** — 현재는 오답 문항 번호를
   칩으로 나열하는 수준이다. 정답 문항까지 포함한 전체 문항 그리드(초록/빨강 배경)로
   확장하면 더 한눈에 들어올 것이다.

## 추후 Phase에서 반영할 항목

1. **알림/푸시 기반 재방문 유도** — "새 테스트 결과가 공개되었습니다" 같은 알림으로
   학부모가 다시 접속하게 만드는 흐름. 현재 알림 시스템은 출결 알림에만 쓰이고 있어
   확장이 필요하다.
2. **학기 단위 성장 곡선(1년 전체 추이)** — 지금은 최근 5회/기간 필터 단위지만, 장기
   재원 학생의 경우 "1학기 대비 2학기" 같은 학기 단위 비교가 학부모 설득에 더 강력할 수
   있다.
3. **PDF/이미지로 리포트 내보내기(공유하기)** — 학부모가 배우자나 다른 보호자에게 리포트를
   공유하고 싶을 수 있다. 현재는 화면 안에서만 확인 가능하다.
4. **목표대학 탭의 실제 변화 이력(시계열)** — 현재는 "최신 상태"만 보여주는 기존
   ParentTargetSummary.tsx로 연결한다. "지난달 대비 어떻게 바뀌었는지" 변화 자체를
   추적하려면 University Analysis 쪽에 스냅샷 이력 저장 기능이 먼저 필요하다.
5. **선생님 공개 코멘트에 학부모 "확인했습니다" 반응 기능** — 현재는 읽기 전용이다.
   학부모가 코멘트를 확인했는지 선생님이 알 수 있게 하면 상호작용이 늘어날 수 있지만,
   "학부모는 확인자다 — 입력 기능 금지" 원칙과 충돌할 수 있어 신중한 검토가 필요하다.

### 트랜치 1에서 실제 반영한 항목

19. **확인 필요한 학생 알림판(위험 신호 자동 산출)** — 신규 순수 모듈
    `src/lib/observationSignals.ts` + 공유 컴포넌트 `src/components/ObservationPanel.tsx`.
    관리자 랜딩(학생목록 상단, 전체 학생 기준)과 선생님 홈(담당 학생 기준)에 강조 패널로
    연결. **선생님 수동 입력 없이** 기존 데이터(공개 테스트 결과 + IF 레코드)만으로
    신호를 계산한다. 표시: 학생명 · 상태 배지 · 감지 이유 칩 · 최근 변화 · [상세 보기] 버튼.
    - 구현한 감지 규칙(신뢰 가능한 것만):
      - `최근 2회 연속 하락` → 배지 **흐름 하락**
      - `최근 3회 중 2회 이상 하위(60% 미만)` → **반복 약점**
      - `직전 시험 반평균 하회` → **확인 필요**
      - `IF 놓친 점수 증가(직전 평균 대비)` → **확인 필요**
      - `최근 테스트 기록 없음/21일 경과` → **학습 리듬 확인**
    - 표현 정책 준수: 배지/이유 문구는 `관찰 필요 · 확인 필요 · 흐름 하락 · 반복 약점 ·
      학습 리듬 확인`만 사용. 금지어(위험/문제 학생, 경고 대상, 탈락, 불합격, 합격
      가능성/합격률 하락) 미사용.

### 트랜치 2로 순연(이번 라운드 미구현 — 다음 라운드 우선)

- **학부모 객관 지표 카드 강화(신규 스펙 #3)** — 평균 대비 위치 · 출결 안정도 · 숙제 수행
  흐름 · IF 놓친 점수 변화 · 좋아진 지표/확인할 지점을 학부모 홈/성장 리포트에 카드로
  확장. 현재 성장 리포트(v3-r3)에 유사 계산이 다수 있어 이를 재사용해 카드화하면 된다.
- **자녀에게 해줄 말 추천(신규 스펙 #4)** — 학생 상태별 자동 문구(점수 하락/‌IF 증가/‌출결
  흔들림/‌상승)를 성장 리포트 하단에 표시. **선생님 수동 입력 없음**, 낙인/압박/불안
  조장 문구 금지. `observationSignals`의 신호를 그대로 재사용해 문구를 매핑하면 로직
  중복 없이 구현 가능하다.
- **자동 브리핑 카드(신규 스펙 #5)** — 학부모 `상담 전 확인 카드`, 선생님 `담당 학생 빠른
  브리핑`. 최근 테스트 변화/출결/숙제/보완 과목/IF + "학부모가 물어볼 질문 추천"을 자동
  조합. 상담 기록 원문은 학부모에게 절대 노출 금지(헌법 §5 유지).
- **출결/숙제 기반 위험 신호 2종** — `결석/지각 증가`, `숙제 미제출 증가`. 학생 단위 출결/
  숙제 집계 헬퍼가 아직 없어, 대리 계산 시 왜곡 위험이 있어 이번엔 넣지 않았다. 다음
  라운드에 `src/lib/studentAttendanceSummary.ts` / `studentHomeworkSummary.ts` 같은
  순수 집계 헬퍼를 먼저 만들고 `observationSignals`에 규칙으로 추가하는 것을 권장한다.
- **시험지별 그래프 심화(구 스펙 #2)** — 내 점수 vs 평균 / 이전 시험 대비 / 문항별 정답률 /
  실제·IF·놓친 점수·상승 가능성 그래프를 학생/학부모/선생님 화면에서 강화. IF 채점은
  성적표 상세 안에서만, IF 이유는 3종(계산 실수/개념 부족/시간 부족)만 유지.
- **테이블 UI 심화 감사(구 스펙 #4)** — 클릭 요소의 버튼/칩화, 보기 전용 텍스트의
  cursor-pointer/hover 제거, 헤더 고정·본문 스크롤·컬럼 폭 정합 전수 점검.

## 입시 인사이트 (후순위 — 구현 보류)

이번 v3-r4에서는 구현 우선순위를 낮춘다(신규 스펙 #6). 후속 구현 시에도 **기사 전문 복사
금지**. 허용 범위는 `제목 / 출처 / 발행일 / 요약 / 외부 링크 / AXIS 해석`으로 한정한다.
가공되지 않은 기사 본문을 저장·표시하지 않으며, 요약과 해석은 자체 문장으로만 작성한다.

### 20. 위험 신호 엔진 확장 (`observationSignals.ts`)

기존 5개 신호(흐름 하락/반복 약점/확인 필요×2/학습 리듬 확인)에 3개를 추가했다.
- **결석/지각 증가** — 최근 14일 결석+지각 건수가 직전 14일보다 많고 2건 이상일 때.
- **숙제 미제출 증가** — 최근 14일 미제출 건수가 직전 14일보다 많고 2건 이상일 때
  (공개된 숙제만 집계).
- **목표 대비 보완 과목 악화** — 신규 `computeSubjectGaps()` 헬퍼로 과목별 목표(90%)
  대비 격차 + 이전 절반 대비 최근 절반 변화를 계산해, 목표 미달 + 3%p 이상 하락한
  과목이 있으면 보고.

배지 표현은 기존 5종(관찰 필요/확인 필요/흐름 하락/반복 약점/학습 리듬 확인)을 그대로
유지했다 — 결석/지각 증가·숙제 미제출 증가·최근 활동 없음은 **학습 리듬 확인**으로,
목표 대비 보완 과목 악화는 **확인 필요**로 매핑했다(신호 종류(kind)는 세분화하되
사용자에게 보이는 배지 어휘는 5종으로 고정).

관리자/선생님 화면 모두 이 확장된 엔진으로 재계산하도록 `TeacherHome.tsx`(담당 학생만)와
`StudentList.tsx`(canAccessStudent 권한 범위 전체 학생)를 갱신했다.

### 21. 자동 브리핑 엔진 (`src/lib/studentBriefingEngine.ts`, 신규)

AI API를 호출하지 않는 규칙 기반 템플릿 엔진. 입력은 `observationSignals`의
`StudentSignalBundle`과 `parentInsightEngine`의 결과뿐이다. 출력 5+1종:
`parentBriefing`(상담 전 확인 카드 본문) · `teacherBriefing`(선생님용 실무 요약) ·
`highlights`(테스트/출결/숙제/보완과목/IF 한 줄 요약 배열) · `checkPoints`(확인할 지점) ·
`parentTalkSuggestion`(자녀에게 해줄 말, 낙인/압박 문구 금지·협력적 어투 고정) ·
`suggestedQuestions`(학부모가 물어볼 질문 추천, 감지 신호 기반 최대 3개).
`parentTalkSuggestion`은 감지된 신호 우선순위(테스트 하락 > IF 증가 > 출결 흔들림 >
숙제 흐름 저하 > 목표대비 악화 > 상승 칭찬 > 기본 격려)에 따라 사전 정의된 문장 중
하나를 고르는 방식이며, 문장 자체를 실시간 생성하지 않는다(전부 정적 템플릿).

### 22. 학부모 객관 지표 엔진 (`src/lib/parentInsightEngine.ts`, 신규)

역시 AI 호출 없는 순수 함수. 8종 산출: 최근 테스트 변화 · 평균 대비 위치 · 출결
안정도(최근 30일) · 숙제 수행 흐름(최근 14일) · IF 놓친 점수 변화 · 목표 대비 보완 과목
(상위 3개) · 좋아진 지표(문장 배열) · 확인할 지점(문장 배열). 각 지표에는 `positive/
neutral/watch` 톤을 부여해 화면에서 색상만 다르게(경고 배지 아님) 표시한다.
Rival/Emblem/SP/Tier 등 게임형 지표는 이 엔진의 어떤 출력에도 없다.
