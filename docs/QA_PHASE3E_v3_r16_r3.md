## 1. 수동 QA 시나리오 — 학생 (`StudentTargetPreview.tsx`)

- [ ] 고3 재원생(모의고사/수능실전 데이터 있음)으로 로그인 → 목표대학 화면 진입
- [ ] 헤더 배지 줄에 `분석 {라벨}` 배지(예: "분석 신뢰도 보통")가 짧게 1개만 보인다
- [ ] 배지 아래 한 줄 문장(`studentMessage`)이 표시되고, 등급 문자(A/B/C/D)·점수·건수
      숫자가 전혀 보이지 않는다
- [ ] 문장에 "합격"/"불합격"/"안정" 관련 표현이 없다 — 담백한 참고 안내 톤인지 확인
- [ ] 데이터가 전혀 없는 학생 계정으로도 확인 → "아직 데이터가 많지 않아요" 류 문구,
      화면이 깨지지 않고 기존 "데이터 없음" 안내와 자연스럽게 공존
- [ ] 기존 "현재 위치"/"목표 변화 가능성"/"추천 적합도"/"보완 필요 과목"/"과목별 상세
      성적 보기(접기)" 섹션이 전부 기존대로 동작한다(순서/내용 변경 없음)

## 2. 수동 QA 시나리오 — 학부모 (`ParentTargetSummary.tsx`)

- [ ] 학부모 계정으로 자녀 선택 → 목표대학/대학추천 화면 진입
- [ ] "설명형 안내" 카드 바로 다음에 **"상담 참고도"** 카드가 새로 보인다
- [ ] 카드 안에는 문장 1개(`parentMessage`)만 있고, 숫자/등급 문자/건수 나열이 없다
- [ ] 문장이 "다음 상담 때 확인해보시길 권장" 같은 설명형 톤인지 확인
- [ ] 자녀가 2명 이상인 계정에서 자녀 전환 시 카드 내용이 올바르게 갱신된다
- [ ] 기존 "데이터 준비 현황"/"밴드 잠금 안내"/"보완 필요 과목 · 다음 상담 포인트"/
      "선생님에게 문의" 섹션이 전부 기존대로 동작한다

## 3. 수동 QA 시나리오 — 선생님 (`TeacherUniversityData.tsx`)

- [ ] 선생님 계정 → 대학추천 데이터 → "상담 요약" 탭 진입
- [ ] "한 줄 요약" 카드 다음에 **"데이터 신뢰도"** 카드가 새로 보인다
- [ ] 카드 상단에 등급 배지(예: "C등급 · 신뢰도 보통")와 근거 설명 문장이 보인다
- [ ] 내신/전국연합모의고사/수능실전모의고사 3개 항목이 **가로 3열 그리드**로 나란히
      배치되어 있다(세로로 쌓여 있지 않은지 PC 화면에서 확인) — 각 항목에 건수와
      얇음/적정/충분 수준 라벨이 보인다
- [ ] 고3 학생 선택 시에만 "수능실전 누적" 안내 줄이 보이고, 고1/고2 학생 선택 시에는
      보이지 않는다(축 자체가 없어야 함)
- [ ] "상담 전 확인 항목"에 부족한 데이터가 우선순위(높음 먼저)대로 나열된다 —
      데이터가 충분한 학생은 "확인할 데이터가 없습니다" 초록 메시지로 대체된다
- [ ] 학생 전환 버튼(가로 스크롤 탭)으로 다른 학생을 선택하면 카드 내용이 갱신된다
- [ ] "상담 원자료 보기" 접기 섹션 등 기존 내용은 그대로 동작한다
- [ ] 내신성적/전국연합/수능실전/데이터 현황 4개 탭은 이번 변경과 무관하게 기존대로
      동작한다(입력 폼 자체는 손대지 않음)

## 4. 수동 QA 시나리오 — 관리자 (`UniversityReportManagement.tsx`)

- [ ] 관리자 계정 → 대학추천/목표대학 관리자 리포트 진입 → 학생 선택("확인" 버튼)
- [ ] 기존 **"상담 준비 상태" 카드 안**에 신뢰도 등급 배지("신뢰도 {문자}등급")가
      기존 데이터 준비 상태 배지 옆에 나란히 추가되어 있다(새 카드가 아님을 확인)
- [ ] 카드 하단에 구분선과 함께 `teacherHeadline` 한 줄(부족 데이터 근거 포함)이
      추가로 보인다
- [ ] "학생 상세에서 상세 분석 확인" 이동 버튼 등 기존 요소는 그대로 동작한다
- [ ] 재원생 목록 화면(학생 선택 전)은 이번 변경의 영향을 받지 않는다(기존대로)
