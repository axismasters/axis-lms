## 1. 엠블럼 배지 재설계 (`AxisEmblemBadge.tsx`)

- **id 고유화(핵심)**: 기존 `uid = iconKey-level-accent` 방식은 같은 종류의 엠블럼이 여러 개
  렌더되면 gradient/filter id가 충돌해 브라우저가 첫 번째 정의만 재사용(색이 뭉개지거나 사라짐)
  하는 문제가 있었다. `useId()`가 생성하는 렌더-인스턴스별 고유 문자열을 접두어로 사용하도록
  변경(`e{useId}`) — **동일 엠블럼 N개 동시 렌더에도 충돌 0**.
- **메달 구조 강화(큰 사이즈 `size >= 88`에서만)**:
  - 링 **리드 눈금** 48개(`RingTicks`) — 동전 테두리 각인 느낌.
  - **하단 리본 플라크 배너**(`PlaqueBanner`) — 참고 엠블럼 시트처럼 메달 아래 매달린 골드
    테두리 네임플레이트 형상(제비꼬리 배너 + 중앙 다이아 장식). 텍스트는 넣지 않고 형상만
    그린다(텍스트는 별도 `AxisEmblemPlaque` 조합).
  - 큰 사이즈는 viewBox를 `0 0 100 118`로 확장하고 높이도 1.18배로(플라크가 잘리지 않게).
  - 작은 배지(20~72px)는 기존과 동일하게 깔끔한 원형 메달만(눈금/플라크 생략 → 뭉개짐 방지).
- **props 시그니처 불변** — 6개 호출부 코드 변경 없음.

## 6. 빌드 검증 관련(중요)

- 이 컨테이너는 npm 레지스트리가 차단(E403)되어 `npm install`/`npm run build`를 로컬 실행할 수
  없다. 그래서 오프라인 스텁 `tsc` 하네스로 **대상 5개 파일 타입 오류 0건 + 전체 회귀 0건**을
  확인했고, `useId`/`React.ReactElement[]` 등 이번 추가분도 통과함을 확인했다.
- **최종 그린 빌드는 GitHub Actions Build Check에서 확정**해야 한다(하네스는 실 `@types`가 아닌
  스텁 기반 근사 검증). r11 산출물이 Actions build를 통과했고, 이번 변경은 그 위에 컴포넌트
  시각/레이아웃 + id 고유화만 얹은 것이라 빌드 리스크는 낮다.

## 7. §GPT(개발 총괄)에게 전달할 의견

1. **`useId()` 도입으로 SVG 인스턴스 id 충돌 계열 버그가 구조적으로 해소됐다.** 앞으로 새
   SVG 컴포넌트를 만들 때도 gradient/filter/clipPath id는 반드시 `useId()` 접두어를 쓰는 것을
   컨벤션으로 삼기를 권한다(하드코딩 id는 리스트/그리드 렌더에서 반드시 충돌).
2. **엠블럼 "detailed" 임계값은 size≥88로 뒀다.** 성장 진열장 대표(100)·MyPage 대표(112)만
   눈금+플라크가 붙고, 갤러리 타일(64~72)·목록(20~48)은 깔끔한 원형만 나온다. 만약 갤러리
   타일에도 플라크를 원하면 임계값을 낮추면 되지만, 64px 이하에선 눈금이 뭉개져 권장하지 않는다.
3. **MyPage/진열장 광폭 기준을 `max-w-[1280px]`로 통일했다.** 다른 학생 화면(홈/성적)도 동일
   기준으로 맞추면 포털 전체 폭감이 일관될 것이다(이번 범위 밖이라 건드리지 않음).
